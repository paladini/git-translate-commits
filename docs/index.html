The Problem

LLMs broke your commit history

AI coding assistants generate commits in whatever language they feel like. Your repo ends up with a messy mix of languages.

How it Works

Three steps. One command.

Scans your repo, detects languages, translates what's needed, and rewrites the history. Done.

🔍

1. Detect

Scans all commits across branches. Detects the language of each message. Skips messages already in your target language.

🌐

2. Translate

Translates messages using an offline neural engine. No API key needed. Preserves Conventional Commits, issue refs, and git trailers.

✅

3. Rewrite

Applies translations to your git history. Creates a backup branch automatically. Files, authors, and dates stay untouched.

Quick Start

Up and running in 30 seconds

Install it, run it, done.

# Install (pick your favorite)
$ pipx install git-translate-commits
$ uv tool install git-translate-commits
$ pip install git-translate-commits
# Preview what will change
$ git-translate-commits --lang en --dry-run

# Translate all commits on current branch
$ git-translate-commits --lang en

# Translate all branches
$ git-translate-commits --lang en --all-branches

# Use LLM for higher quality (optional)
$ git-translate-commits --lang en --engine llm
